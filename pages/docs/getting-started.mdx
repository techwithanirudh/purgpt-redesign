### Unleashing the power: using the AI

Once you have your key, the power of PurGPT can be unleashed. Before moving to this step, if you wish, you may test out PurGPT by chatting with it: Use the `/chat` command!

The way that one interacts with PurGPT is, currently, through POST requests (whether that be through cURL, `fetch()`, or other means). This makes it easy to use and cross platform.

A simple demonstration of using PurGPT, using the `completions` model, is below:

   ```js copy filename="index.js"
    const http = require("https");

    const url = 'https://purgpt.xyz/v1/chat/completions';

        method: 'POST',
        'Content-Type': 'application/json',
        'Authorization': 'Bearer YOUR API KEY',

    });

    let result = '';
    const req = http.request(url, options, (res) => {
        console.log(res.statusCode);

        res.setEncoding('utf8');
        res.on('data', (chunk) => {
            result += chunk;
        });

        res.on('end', () => {
            console.log(result);
        });
    });

    req.on('error', (e) => {
        console.error(e);
    });

    req.write(data);
    req.end();
    ```

### Understanding the power: how to use PurGPT

PurGPT is simple: all you have to do is send a POST request to a URL which, depending on your desired company (openai) and product (completions), will give you your desired output.

Of course, you can't possibly be able to read our minds, so the URLs, as well as their descriptions, will be described in the next sections.

Another thing to note is that PurGPT request bodies match somewhat __exactly__ (we have certain things that we provide that you can add on, read the docs to find out what!) the same thing that you would send over to OpenAI's API. This means that you can always use their [API Reference](https://platform.openai.com/docs/api-reference/introduction) to find information about what should be in the request body.
